# Qt基础

Qt是一个开发C++可视化程序的基础框架。

在 Qt 中，绝大多数自定义窗口都是继承自某个 Qt 基类。这意味着你的窗口不仅仅是一个 C++ 类，它直接继承了 Qt 庞大的窗口系统能力（如绘制、事件处理、窗口句柄管理）。

**Qt 的核心魔法：Q_OBJECT 宏**

```c++
    Q_OBJECT
```

- **Qt 核心原理 (元对象系统 / Meta-Object System)**:
  - **问题**: 标准 C++ 是静态语言，缺乏**反射 (Reflection)** 和 **内省 (Introspection)** 机制（即程序在运行时无法知道自己有哪些方法、属性，类名是什么）。
  - **解决方案**: Qt 发明了 **MOC (Meta-Object Compiler)**。在编译你的 C++ 代码之前，`moc` 程序会先扫描所有包含 `Q_OBJECT` 宏的文件。
  - **产物**: `moc` 会自动生成一个对应的 `.cpp` 文件（通常叫 `moc_MainWindow.cpp`）。这个文件里包含了大量的元数据（Metadata），<span style="color:#d59bf6;">描述了你的类有哪些信号、哪些槽、类名是什么等等。</span>
  - **作用**: **没有这个宏，信号与槽（Signals & Slots）机制就无法工作**，你也无法使用 `tr()` 进行国际化翻译，也无法使用 Qt 的属性系统。

**Qt 核心原理 (对象树 / Object Tree)**:

- 注意参数 `QWidget *parent = nullptr`。这是 Qt 内存管理的核心。
- **设计哲学**: Qt 中的对象（QObject 及其子类）可以形成父子关系树。
- **内存自动回收**: 当一个**父对象**（Parent）被析构（销毁）时，它会自动 `delete` 它所有的**子对象**（Children）。
- **用法**: 对于 `MainWindow`，`parent` 默认为 `nullptr`，表示它是顶层窗口，没有父窗口。当你在这个窗口里创建按钮时，你会把 `this` (MainWindow) 传给按钮作为 parent，这样当 MainWindow 关闭时，按钮会被自动销毁，防止内存泄漏。

**Qt 核心原理 (信号与槽)**:

- **关键字 `slots`**: 这是一个 Qt 特有的扩展关键字（实际上也是宏）。它告诉 MOC 编译器：**“下面的函数不仅是普通的 C++ 成员函数，还是一个可以在运行时被动态调用的‘槽’。”**

- **底层机制**:

    1. **信号 (Signal)**: 当用户点击按钮时，`QButtonGroup` 会“发射”一个信号。例如 `signal: buttonClicked(int id)`。这本质上是一个函数调用，但它不知道谁会接收它。

    2. **槽 (Slot)**: `switchPage(int id)` 是接收者。

    3. **连接 (Connect)**: 在 `.cpp` 文件中，你会写类似这样的代码：

        C++

        ```c++
        // 伪代码：将 导航组 的 点击信号 连接到 主窗口 的 切换页面槽
        connect(navGroup, &QButtonGroup::idClicked, this, &MainWindow::switchPage);
        ```

    4. **调用过程**: 当按钮点击发生 -> Qt 事件循环捕获 -> 触发信号 -> MOC 查找元数据表 -> 找到连接的槽函数地址 -> 调用 `switchPage(id)`。

- **优势**: 相比 C 语言的**回调函数 (Callback)**，Qt 的信号槽是**类型安全**的（参数类型必须匹配），且完全**解耦**（发射信号的人不需要知道谁在接收）。

在 Qt 的世界里，你看到的任何一个按钮、输入框、标签，甚至整个窗口，本质上都是一个 `QWidget`

它的核心角色可以概括为以下几点：

1. UI 的基本原子

`QWidget` 是用户界面的最小构建单元。它有两个最基本的超能力：

- **画自己**：它知道如何在屏幕上绘制自己（处理形状、颜色等）。
- **听指令**：它能接收并处理用户的各种输入事件，比如鼠标点击、键盘按键。
- 万物之母（基类）

几乎所有你认识的 Qt 控件都是继承自 `QWidget` 的：

- **QPushButton**（按钮）是 QWidget。
- **QLabel**（标签）是 QWidget。
- **QLineEdit**（输入框）是 QWidget。
- 甚至你代码里的 **QMainWindow**（主窗口）也是从 QWidget 派生出来的。
- 容器与窗口

- **作为容器**：它可以像一个盒子一样，里面装满其他的按钮或输入框。
- **作为窗口**：如果一个 `QWidget` 没有“爸爸”（parent 为空），它就会自动变成一个独立的、可以移动和缩放的操作系统窗口。
- 内存管家

正如你在代码中看到的 `parent` 参数，`QWidget` 建立了 Qt 的**父子对象机制**。只要你给子控件指定了父控件，当父控件被销毁时，所有子控件都会被自动清理，防止内存泄漏。

**一句话总结：**
如果你要把某个东西显示在屏幕上并让它能响应操作，在 Qt 里你首先就会想到 `QWidget`。
