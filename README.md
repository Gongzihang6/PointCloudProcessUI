# 🐖 Multi-Camera 3D Pig Body Measurement System

**(多相机 3D 猪只体尺非接触式测量系统)**

## 📖 1. 项目简介与背景意义

在现代智慧农业与生猪养殖中，精确获取猪只的体尺参数（体长、体高、体宽、胸围、腰围、臀围）对于评估生长发育状态、饲料转化率以及遗传育种具有极其重要的意义。传统的接触式测量方法费时费力，且容易引起动物应激反应。

本项目研发了一套**基于多相机（5个 Femto Bolt / Azure Kinect）的 3D 点云非接触式体尺测量桌面端软件**。通过融合多视角的 3D 点云数据，结合工业级点云处理算法与前沿的深度学习关键点检测模型（Pointcept/OctFormer），实现了从**原始数据采集 -> 预处理与配准 -> 主体精细提取 -> 关键点定位 -> 体尺自动计算 -> 报告导出**的全自动流水线。

## 🏗️ 2. 核心技术选型与架构设计

为了彻底解决 Windows 环境下 C++ 与复杂深度学习算子（如稀疏卷积、八叉树）编译冲突的“地狱级”难题，本项目采用了创新的 **C/S (Client-Server) 前后端分离架构**。

## 🖥️ 客户端 (前端 / Windows)

-   **技术栈**: C++ 17, MSVC, Qt 6, PCL (Point Cloud Library) 1.14, VTK
-   **职责**: 负责 UI 交互、点云海量数据的高性能 3D 渲染、基础点云算法（滤波、降采样、ICP配准、RANSAC 平面拟合、欧式聚类）、高阶几何计算（PCA 对齐、样条插值周长计算）。
-   **特色**: 接入原生 PCL/VTK 交互，实现了类似 *CloudCompare* 的专业级鼠标操控体验（右键平移、左键旋转、Shift+左键精准拾取）。

## ⚙️ 服务端 (后端 / WSL2 Linux)

-   **技术栈**: Python 3.10, PyTorch, Pointcept, FastAPI
-   **职责**: 常驻 GPU 显存，加载训练好的 3D 深度学习模型（OctFormer）。提供 HTTP Web 接口。
-   **通信机制**: 前端将 $N \times 7$ (包含曲率和法线) 的点云特征序列化为紧凑的**二进制字节流**，通过 `multipart/form-data` 发送至后端，后端推理后以二进制坐标流返回，实现毫秒级的零拷贝通信。

## 🧠 3. 系统设计逻辑与算法流水线

本软件的数据处理流水线严格遵循以下 5 个模块：

1.  **预处理与裁剪 (Preprocess)**
    -   对 5 个相机的原始点云分别执行 VoxelGrid 降采样。
    -   执行 SOR (Statistical Outlier Removal) 统计滤波去除飞溅噪点。
    -   使用 PassThrough/Radius 裁剪限定有效视野。
2.  **配准与融合 (Registration)**
    -   加载预先标定好的 $4 \times 4$ 齐次变换矩阵（`double` 精度存储，防止大平移量带来的精度丢失）。
    -   将 LB, LT, RB, RT 四个视角的点云统一变换至 Top 顶视相机坐标系下并进行拼接。
3.  **主体精细提取 (Extraction)**
    -   **过河拆桥策略**: 首先使用迭代 RANSAC 算法拟合提取最大平面（地面/称重台）并强制剔除，切断猪蹄与地面的物理连接。
    -   使用 KD-Tree 加速的欧式聚类 (Euclidean Cluster Extraction) 提取点数最大的连通簇作为猪只主体。
    -   执行 MLS (Moving Least Squares) 表面平滑，使猪体表面犹如丝绸般光滑。
4.  **关键点检测与测量 (Keypoints & Measure)**
    -   **AI 自动检测**: 提取干净的背部点云，计算表面法线与曲率，发送至 WSL2 后端，推理获取 P1-P6 关键点。
    -   **手动介入 (Human-in-the-loop)**: 支持按住 `Shift + 左键` 在 3D 空间中连续手动拾取补充关键点。
    -   **体尺计算**:
        -   使用 PCA 主成分分析将猪体对齐至坐标轴。
        -   计算背部表面骨架 (Skeletonization) 用于测定测地线体长。
        -   使用法向量切片截取胸/腰/臀部轮廓，并通过 Catmull-Rom 3D 样条插值闭合多边形，计算精准周长。
5.  **数据导出 (Export)**
    -   支持一键保存融合点云与干净的主体点云为 `.pcd` 二进制文件。
    -   自动生成并导出排版精美的 `.txt` 测量报告。

## 🚀 4. 环境配置与部署指南

## 4.1 服务端部署 (WSL2 / Ubuntu)

1.  克隆 Pointcept 框架并配置 Conda / uv 虚拟环境。

2.  安装 PyTorch 及对应的 CUDA 工具链。

3.  编译自定义算子（spconv, pointops）。

4.  运行 FastAPI 服务：

    ```bash
    cd tools/
    uvicorn KeyPointPrediction_Qt:app --host 0.0.0.0 --port 8000
    ```

## 4.2 客户端编译 (Windows)

1.  安装 Visual Studio 2022 (包含 C++ 桌面开发工作负载)。

2.  安装 Qt 6.x，确保勾选 MSVC 2019/2022 编译器。

3.  安装 PCL 1.14.1 及其第三方依赖 (VTK, Eigen, Boost, FLANN)。

4.  配置环境变量 `PCL_ROOT` 等。

5.  使用 CMake 生成 VS 解决方案并编译：

    ```bash
    mkdir build && cd build
    cmake .. -G "Visual Studio 17 2022" -A x64
    cmake --build . --config Release --parallel 8
    ```

## 🖱️ 5. 软件使用指南

本软件分为**单体模式 (Single Mode)** 和 **批量模式 (Batch Mode)** 两个主要工作流，分别用于算法参数调试与高通量无人值守处理。

### 5.1 🕹️ 通用 3D 交互操作 (基于 VTK/PCL)

在任何模式的 3D 视图窗口中，您都可以使用以下工业级标准的鼠标交互操作：

-   **旋转视角**：按住 `鼠标左键` 拖拽。
-   **平移视图**：按住 `鼠标右键` 拖拽 (CloudCompare 风格)。
-   **缩放视角**：滚动 `鼠标滚轮`。
-   **设置旋转中心**：`左键双击` 点云上的任意部位。系统会利用底层深度缓冲 (Z-Buffer/CellPicker) 精准拾取该点，并在控制台打印坐标，后续的左键旋转将完美环绕该点进行。

------

### 5.2 🐖 单体模式 (Single Mode) —— 用于参数调试与单次精细测量

单体模式提供了最细粒度的控制，适合对异常数据进行人工微调或寻找最佳算法参数。

#### Step 0: 导入数据与图层控制

1.  **加载文件**：可以直接点击左侧面板各相机的 `📂` 图标逐个加载 `.pcd` 文件；或者点击 **“📥 加载文件夹...”**，系统会自动扫描所选文件夹并根据文件名后缀（如 `_d_pc.pcd`）自动对齐加载 Top, LB, LT, RB, RT 五个视角的点云。
2.  **图层切换**：左侧的“视图图层”面板提供了如同 Photoshop 般的独立图层开关。您可以随时勾选/取消勾选来查看特定相机的数据，或者查看“✨ 融合后点云”、“🐷 提取主体”和“📏 体尺测量结果”等中间结果。

#### Step 1: 预处理与裁剪 (Preprocess)

1.  调整下采样体素、SOR 统计滤波参数及背景裁剪半径。
2.  点击 **“👁️ 预览效果”**，在 3D 视图中观察滤波情况。
3.  满意后点击 **“🛠️ 执行处理”**，将清洗后的点云覆盖至系统内存。

#### Step 2: 配准与融合 (Registration)

1.  选择 **配准算法**：支持 `手动矩阵`、`ICP (点到点/点到面)` 以及 `NDT 配准微调`。切换算法时，对应的参数设置面板会自动展开。
2.  **矩阵重置与编辑**：若对算法自动对齐的结果不满意，随时可以点击 **“⟲ 重置”** 恢复为出厂物理标定矩阵。
3.  **多线程执行**：勾选需要配准的源相机，点击 **“🚀 执行配准与融合”**。系统将启动多线程后台并发计算，UI 不会卡顿，完成后会自动生成带原生 RGB 颜色的融合点云。

#### Step 3: 主体精细提取 (Extraction)

1.  设置“地面滤除厚度”以斩断猪蹄与地面的物理连接。
2.  设置聚类容差和最小点数，点击 **“🐷 提取平滑最大主体”**。系统会自动将环境背景剥离，并在图层中高亮显示纯净的粉色猪体。

#### Step 4: 关键点与测量 (Measure)

1.  **AI 自动检测**：默认处于 AI 模式，点击 **“⚡ 运行模型推理”**，系统会提取背部特征发送至后端，几秒内完成 6 个关键点的检测，并在下方的状态指示灯面板亮起绿灯。
2.  **人工介入 (手动拾取)**：若 AI 检测偏差，可切换至 **“👆 手动拾取”** 模式。根据控制台提示，按住 `Shift + 鼠标左键`，依次在 3D 空间中点选 P1~P6。
3.  点击 **“📏 计算体尺参数”**。系统将瞬间完成切片计算，并在左下角生成并激活“📏 体尺测量结果”图层（包含绿色骨架、彩色周长线圈和高宽包围盒）。

#### Step 5: 导出数据 (Export)

1.  **💾 另存点云**：可将“融合点云”与“主体点云”单独保存为 `.pcd` 格式供后续复用。
2.  **📸 导出测量可视化**：将计算好的 3D 线框渲染为带有**物理管径厚度**的彩色点云保存。在 CloudCompare 中打开即可获得与软件内完全一致的粗壮可视化线条。
3.  **📄 导出测量报告**：一键生成排版精美的 `.txt` 格式工业测量报告，无乱码风险。

### 5.3 📂 批量模式 (Batch Mode) —— 用于高通量无人值守处理

批量模式利用多核 CPU 与异步调度，能够稳定连续地处理成百上千个时间戳文件夹。

1.  **配置 IO 目录**：
    -   **输入目录 (Input)**：选择包含多个时间戳子文件夹（如 `20251102_115330_818`）的总目录。每个子文件夹内需包含相应的视角点云文件。
    -   **输出目录 (Output)**：选择用于保存处理结果的空白目录。
2.  **配置全局参数**：
    -   在中间的滚动视图中，一次性设置好适用于该批次硬件环境的预处理、配准 (推荐使用默认或 ICP 微调)、主体提取和体尺测量参数。
3.  **开始运行**：
    -   点击 **“🚀 开始批量处理”**。
    -   系统将启动**防 OOM (内存溢出) 机制**，以串行方式遍历文件夹，而在处理单个文件夹内部相机时开启多核并发。
    -   控制台会实时打印带颜色的时间戳日志，底部的进度条会同步推进。
4.  **安全中止 (Graceful Shutdown)**：
    -   如果通过日志发现参数设置有误（如点云被全部滤空），点击 **“🛑 停止处理”**。
    -   系统不会粗暴杀掉进程导致崩溃，而是会耐心等待当前正在处理的文件夹安全落盘后，平稳退出批处理循环。
5.  **查看批处理结果**：
    -   处理完成后，前往您的 Output 目录。里面会自动生成与 Input 同名的时间戳文件夹。
    -   每个成功的文件夹内都会包含 4 个自动生成的文件：`Merged_Cloud.pcd` (融合点云)、`Pig_Body.pcd` (主体点云)、`Measurement_Visual.pcd` (线框可视化彩色点云) 以及 `Measurement_Report.txt` (格式化数据报告)。

## ⚠️ 6. 注意事项与常见问题 (Troubleshooting)

-   **单位统一**: 整个系统中所有的坐标、参数和结果均统一采用点云自带的 **毫米 (mm)** 作为单位。
-   **UI 卡顿**: 虽然绝大部分计算非常迅速，但大点云的 MLS 平滑和体尺计算可能需要几秒钟。已在底层剥离了可视化循环 (`spinOnce`) 避免 Qt 主线程死锁。
-   **HTTP 请求失败**: 如果点击“运行模型推理”提示 `Internal Server Error` 或连接失败，请检查：
    1.  WSL2 中的 Python 服务是否已启动。
    2.  WSL2 的 IP/端口端是否能被宿主机访问（可尝试关闭 Windows 防火墙或检查 127.0.0.1 映射）。
-   **地面未被彻底去除**: 猪体聚类失败通常是因为猪蹄和地面连在一起。请在 UI 中适当调大 **“地面滤除厚度 (mm)”** 参数后再点击提取。
